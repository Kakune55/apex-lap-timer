# Apex Lap Timer - 设计系统与风格指南

本文档以当前实现为准，定义 Apex Lap Timer 的统一视觉语言。后续新增页面和组件应复用本文中的 token 与语义类，不再引入平行风格。

---

## 1. 核心审美
- **定位：** 高性能硬件感（专业遥测终端 + 驾驶舱 HUD）。
- **目标：** 数据可读性第一，装饰第二。
- **体验关键词：** 精确、克制、沉浸、低干扰。

---

## 2. 色彩与 Token

### 2.1 核心颜色

| 语义 | Token | 值 | 用途 |
| :--- | :--- | :--- | :--- |
| 背景 | `--bg-color` | `#0a0a0a` | 应用主底色 |
| 卡片底 | `--card-bg` | `#151619` | 组件容器基底 |
| 表面 1 | `--surface-1` | `#101115` | 次级容器/深层卡片 |
| 表面 2 | `--surface-2` | `#1a1c21` | 高亮卡片渐变顶部 |
| 主文本 | `--text-primary` | `#FFFFFF` | 标题、核心数值 |
| 次文本 | `--text-secondary` | `#8E9299` | 标签、描述文案 |
| 强调绿 | `--accent-green` | `#00FF66` | 主操作、正向状态、速度 |
| 强调红 | `--accent-red` | `#FF3333` | 警示、减速、负向状态 |
| 强调蓝 | `--accent-blue` | `#3B82F6` | 横向 G 力、趋势辅助 |
| 强调黄 | `--accent-yellow` | `#FBBF24` | 总 G、综合载荷 |

### 2.2 背景氛围
- 页面背景使用多层渐变（深灰线性 + 绿/蓝径向弱光），禁止纯平单色黑底。

---

## 3. 字体规范
- **无衬线主字体：** `Inter`, `PingFang SC`, `Hiragino Sans GB`, `Microsoft YaHei`, `Heiti SC`, `sans-serif`。
- **数值字体：** `JetBrains Mono` 优先，所有实时数字必须 `tabular-nums`。
- **标签文案：** `text-[10px]` + `font-bold` + `uppercase` + `tracking-widest`。
- **大数值：** `font-bold` + `tracking-tighter`，避免轻细字重。

---

## 4. 语义样式类（强制复用）

以下类已在 `src/index.css` 实现：

| 类名 | 作用 |
| :--- | :--- |
| `apex-label` | 通用小标签样式 |
| `apex-panel` | 主要卡片（渐变表面 + 边框 + 深阴影） |
| `apex-panel-muted` | 次级卡片/弱信息容器 |
| `apex-glass` | 毛玻璃浮层（状态条、悬浮控件） |
| `apex-pill` | 胶囊浮层（头部按钮、地图模式标签） |
| `apex-btn-primary` | 主按钮（强调绿） |
| `apex-btn-secondary` | 次按钮（半透明深色） |

规则：
- 新组件优先使用上述语义类，避免重复写 `bg-* + border-* + blur-*` 组合。
- 主按钮默认绿色，不再使用白底黑字作为主 CTA。

---

## 5. 地图与图表

### 5.1 地图
- 瓦片统一为深色：`CartoDB dark_all`。
- 参考线为低透明白线，当前圈轨迹高对比显示。
- 门线语义：起点绿、终点红、分段黄。

### 5.2 图表
- 图表库：`Recharts`。
- 面积图渐变使用语义色 token，不直接硬编码十六进制。
- 网格线统一：`rgba(255,255,255,0.05)`。
- 保留 `Tooltip + Brush + syncId` 的联动分析模式。

---

## 6. 交互与动效
- 动效框架统一：`motion/react`。
- 控件交互反馈：优先亮度变化，其次轻微缩放（`scale`）。
- 浮层与弹窗动画保持短时、快速响应，避免夸张位移。

---

## 7. 一致性检查清单
- 是否复用语义类（`apex-panel`/`apex-btn-primary` 等）。
- 实时数字是否全部 `tabular-nums`。
- 是否避免新增非语义硬编码颜色。
- 地图是否保持深色底图。
- 主操作按钮是否仍为强调绿体系。
